# Integral Decisions

This repository is the append-only governance record of the Integral project. 
Every ratified decision that affects the architecture, a foundational principle, 
an interface contract, or the governance of the project itself is recorded here 
as a Decision Record.

Decision Records are immutable once ratified. They are never edited, only 
superseded by new decisions that reference the original. This repository is 
the project's institutional memory — the record that allows future contributors 
to understand not just what the system is, but why it is that way.

## Decision Record Format

Decision Records are named `DR-NNN-brief-title.md` and contain:

- **Decision** — what was decided
- **Context** — the problem or question that prompted the decision
- **Alternatives considered** — other approaches that were evaluated
- **Reasoning** — why this approach was chosen over the alternatives
- **Participants** — who was involved in the deliberation
- **Date** — when the decision was ratified
- **Supersedes** — reference to any prior decision this replaces, if applicable

## How Decisions Are Created

Decision Records are created automatically when a pull request is merged in 
any Integral repository following ratification through the standard governance 
workflow. The creation process is described in Section 6.5 of the development 
guide.

A decision does not exist in Integral without a traceable record of how it 
was reached. This is true from the first day of operation.
